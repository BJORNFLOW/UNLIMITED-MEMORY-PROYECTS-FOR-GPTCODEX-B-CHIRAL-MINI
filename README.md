# CHIRAL MEMORY MINI  
## Project memory for Codex

**We do not replace Codex. We give it continuity.**

At **Bjorn Team**, we developed Chiral Memory Mini, a Windows application that works alongside Codex without replacing its interface or bypassing its permission controls.

Tools such as OpenClaw and Hermes provide their own experiences for working with AI agents. We chose a different path.

Codex already delivers the combination we wanted: precision, speed, advanced tools and an intuitive interface. We did not want users to leave Codex or learn an entirely different platform.

Our previous application, T-REX, explored the idea of building a complete environment around AI. That experience taught us something important:

**Users do not need another interface. They need Codex to remember their projects better.**

That is why we created **Chiral Memory Mini**.

Chiral connects Codex to local memory associated with the project. It stores and organizes decisions, changes, context and versions. When a new task begins, it retrieves the relevant information so Codex can continue working with better project context.

This makes it possible to recover a decision made months earlier, locate the related code and understand why it was implemented that way, without explaining everything again from the beginning.

## How does it work?

**Memory is automatic: simply install Chiral and open your project in Codex.**

Then continue using Codex as usual. Chiral works around the project, keeps its memory inside the project folder and prepares the context needed for future tasks.

The information remains local, readable, verifiable and under the user’s control.

Chiral is not another artificial intelligence and it does not attempt to replace Codex. Its purpose is to enhance it: preserving continuity across long projects, reducing forgotten details and helping every new task begin with the right context.
